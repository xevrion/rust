## Overview  

`xevrion/rust` is a tiny repository that demonstrates the canonical layout of a Rust project, featuring a classic “Hello, world!” program. It showcases:

* **Zero‑dependency setup** – only the Rust toolchain is required.  
* **Idiomatic project structure** – `src/` (here `hello_world/`) with a `main.rs` entry point.  
* **Ready‑to‑extend foundation** – add crates, modules, or binaries without restructuring.

> **Target audience** – Rust newcomers, educators, and developers who need a clean starting point for new command‑line tools or libraries.

Current version: **0.1.0** (initial release).

## Architecture  

```
xevrion/rust/
├─ .gitignore          # Standard Rust ignore rules
├─ README.md           # This documentation
└─ hello_world/        # Example binary crate
   ├─ Cargo.toml       # (generated by `cargo init --bin`)
   └─ src/
       └─ main.rs      # Entry point – prints “Hello, world!”
```

*The repository follows the default Cargo layout for a binary crate.*  
- **`Cargo.toml`** (generated on first `cargo init`) declares the package name, version, and edition.  
- **`src/main.rs`** contains the executable logic.  
- Adding more binaries or libraries simply involves creating additional directories under the workspace root and referencing them in the workspace `Cargo.toml`.

## Getting Started 🚀

### Prerequisites  

| Tool | Minimum version | Installation |
|------|-----------------|--------------|
| **Rust toolchain** | `1.74` | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh` |
| **Git** | any | <https://git-scm.com/downloads> |
| **Optional: Docker** | `20.10` | <https://docs.docker.com/get-docker/> |

> **Note** – The project compiles on any platform supported by Rust (Linux, macOS, Windows).

### Installation  

To get started with one of the example projects, navigate into its directory:

bash
cd guessing_game


# or
cd hello_world

# 1️⃣ Clone the repository
git clone https://github.com/xevrion/rust.git
cd rust

## 2️⃣ (First‑time only) Initialise the Cargo project if missing 🛠️

This step is only necessary if you are creating a new Rust project from scratch or converting a standalone `.rs` file into a Cargo project.

For example, if you were to convert the `hello_world` example into a Cargo project:

bash
cd hello_world
cargo init --bin .


> **Note**: The `hello_cargo` example project already contains a `Cargo.toml` file and does not require this initialization step.
#    This will create Cargo.toml and the src folder.
cargo init --bin hello_world

# 3️⃣ Build the binary
cargo build --release   # produces target/release/hello_world

# 4️⃣ Verify the build
./target/release/hello_world
# Expected output:
# Hello, world!
```

### Configuration  

The example does not require any runtime configuration.  
For future extensions, you may add an `.env` file and read variables with the `dotenv` crate:

```dotenv
# .env (example)
GREETING="Hello, Rustaceans!"
```

## Usage  

Run the compiled binary directly or via Cargo:

bash
# Navigate to the guessing game directory
cd guessing_game

# Run the game using Cargo
cargo run


## Using Cargo (recommended during development) 📦

To run a specific Cargo project, navigate into its directory and use `cargo run`.

For example, to run the `hello_cargo` project:

bash
cd hello_cargo
cargo run --quiet


This will compile and execute the `main.rs` file within the `hello_cargo` project.
# Using the compiled binary
./target/release/hello_world
```

**Sample output**

```
Hello, world!
```

### Extending the program  

Replace the contents of `src/main.rs` with your own logic.  
For example, to accept a command‑line argument:

```rust
use std::env;

fn main() {
    let name = env::args().nth(1).unwrap_or_else(|| "world".into());
    println!("Hello, {}!", name);
}
```

Run:

```bash
cargo run -- Alice
# → Hello, Alice!
```

## Deployment  

For simple command‑line tools, distribution can be performed via:

1. **Pre‑compiled binaries** – `cargo build --release` and ship the binary.  
2. **Docker container** – see the Dockerfile example below.

### Docker (optional)

```dockerfile
# Dockerfile
FROM rust:1.74-alpine AS builder
WORKDIR /app
COPY . .
RUN cargo build --release

FROM alpine:latest
COPY --from=builder /app/target/release/hello_world /usr/local/bin/hello_world
ENTRYPOINT ["hello_world"]
```

Build & run:

```bash
docker build -t hello_world .
docker run --rm hello_world
```

## Troubleshooting  

| Problem | Solution |
|---------|----------|
| `rustup: command not found` | Install Rust via the official script (see Prerequisites). |
| Build fails with `cannot find crate` | Ensure you are inside the repository root and that `Cargo.toml` exists. Run `cargo clean && cargo build`. |
| Binary prints nothing | Verify you are running the compiled binary (`./target/release/hello_world`). |
| Docker build fails on Alpine | Install required build tools: `apk add --no-cache musl-dev`. |

For further help, open an **Issue** on GitHub.
